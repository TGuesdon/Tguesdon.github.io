Professionnal project ( Idealtek Concept )
TLDR :

MenuLib is a web application which allows customers to order at the restaurant with a Qr Code.

Technologies used :
  • Angular
  • Firebase

MenuLib is a web application which allows customers to order at the restaurant with a Qr Code. Its main difference with similar application is that the Qr Code is displayed on a screen and changed after each utilisation. This protect toward any malicious possibility.

Project Manager

My first role and this project was a supervisor role, I helped Idealtek Concept to write their specifications and made contact with Paul, Louis and Thomas. I had to verify that they completed our expectation during the project, and was their main contact at Idealtek Concept.

They delivered the application in time with all the asked functionnalities.

Functionnalities V1 :

Restaurant admin panel

A dashboard only accessible by the restaurant administrator

  • Add / Edit / Delete articles
  • Add / Edit / Delete categories
  • Add "meal of the day" articles
  • Theme choice
  • Restaurant settings
  • Add / Edit / Delete security code
Admin article view
Admin settings view

Preparer panel

A view of all orders being prepared, which possibility to validate and clear them. Used in the kitchen restaurant.

Customer panel

A view used to browse all articles and categories, with a cart that can be validated so the order goes to the kitchen.

Developper

After getting the V1 from Louis, Paul and Thomas, I switched back to a developper role. We showed our application to some potential client to get some insights on what to improve next.

I did a full visual reworking to get a more "sellable" visual, with the addition of new themes for caf√© and pizzeria. This is also when we decided to use a Qr Code displayed on screen that change after each use. I added this functionnality along with a number id for the orders.
